# Палитра веток (13 цветов) — заметки

Источник данных (`02_Branches`) описывает цвет каждой ветки словом (синий,
голубой, жёлтый, золотой, серебряный, зелёный, зелёный тёмный, фиолетовый,
оранжевый, красный, серый, радужный, бирюзовый) — не hex-кодом. Ниже — как эти
слова превращены в конкретные hex-значения в `scripts/convert.py`.

## Метод

Цвета сгенерированы в OKLCH (лёгкость/хрома/оттенок), чтобы держать одинаковую
лёгкость и насыщенность у всех 13 слотов, затем провалидированы скриптом из
dataviz-skill (`validate_palette.js`) по чек-листу: полоса лёгкости, порог
хромы, CVD-разделение (Machado 2009, все пары — так как это карта, где любые
два маркера могут оказаться соседями), контраст к поверхности.

## Финальная палитра

| Ветка | Слово из xlsx | Hex |
|---|---|---|
| Оптика / камера-обскура | синий | `#1f7dcf` |
| Оптические инструменты | голубой | `#009ca5` |
| Фотохимия | жёлтый | `#9b7e00` |
| Гелиография | золотой | `#bc8900` |
| Дагеротип | серебряный | `#089ac3` |
| Фотогенный рисунок / фотограмма | зелёный | `#438f31` |
| Негатив-позитив | зелёный тёмный | `#00724c` |
| Фиксаж / терминология | фиолетовый | `#8059bb` |
| Печать / тиражирование | оранжевый | `#ba2d1f` |
| Мокрый коллодий | красный | `#c13a46` |
| Уникальный позитив | серый | `#006995` |
| Цвет | радужный | `#ae539d` |
| Проекция / распространение | бирюзовый | `#00988a` |

## Результат валидации

Против тёмной поверхности карты (`#020B18`, бренд Navy) и светлой поверхности
панелей (`#F7F5EF`, бренд Warm Paper), все пары (`--pairs all`):

- Полоса лёгкости (OKLCH L): PASS для всех 13.
- Порог хромы (C ≥ 0.10): PASS для всех 13.
- Контраст к поверхности ≥ 3:1: PASS почти везде; 1 предупреждение (WARN) на
  светлой поверхности для «Гелиография» (2.87:1) — смягчается тем, что цвет
  никогда не используется без подписи (легенда, чипы фильтров, карточка узла).
- CVD-разделение (протан/дейтан/тритан): 12 из 13 цветов разделены с запасом
  (ΔE существенно выше целевых 12). Одна пара — «Цвет» (`#ae539d`) и
  «Уникальный позитив» (`#006995`) — не проходит порог при протанопии (ΔE 3.3).
  13 категорий превышают документированный безопасный потолок в 12 цветов;
  это единственная оставшаяся коллизия после нескольких итераций подбора.

## Почему это допустимо

Каждый маркер и каждая ветка **всегда** сопровождаются текстом (легенда, чипы
фильтров, всплывающая подсказка, карточка узла) и независимой формой маркера
(тип узла — 11 форм, не зависит от цвета). Цвет никогда не единственный
носитель информации, поэтому оставшаяся CVD-коллизия одной пары не приводит к
потере данных для пользователей с цветовой слепотой — это ровно то смягчение
(secondary encoding), которое требует методика при 13 категориях.

## Если добавится 14-я ветка

Конвертер не придумывает цвет сам — печатает предупреждение в stderr. Нужно:
1. Выбрать новый hex вручную (или через тот же OKLCH-подход).
2. Добавить его в `BRANCH_COLORS` в `scripts/convert.py`.
3. Прогнать `validate_palette.js` заново на всех 14 цветах (`--pairs all`,
   оба режима — light/dark) и по возможности сдвинуть цвета с худшим ΔE.
