Мобильная адаптация Explore Mode (≤768px). Journey Mode и десктоп не трогать.

## Главная проблема

Explore Mode на мобильном показывает десктопный layout: карта — узкая полоска слева (~80px), правая панель занимает остаток ширины. Это неработающий десктопный flex-row на маленьком экране. Плюс огромное пустое белое пространство под шкалой времени.

## Layout на мобильном

Вертикальный layout (flex-column), сверху вниз:

1. **Шапка** — компактная, максимум 44px. Убери заголовок «Историческая карта фотографии» на мобильном (оставь только «PHOTO LAB BY VF» мелко + переключатель справа). Кнопка «Фильтры» и счётчик «68 узлов» — в отдельной строке под шапкой.

2. **Карта** — на всю ширину, высота ~50vh. Тёмная тема карты уже есть — оставить.

3. **Правая панель** (карточка узла + глобальные процессы) — под картой, на всю ширину, высота по контенту с overflow-y: auto. Не рядом с картой, а под ней.

4. **Шкала времени** — под правой панелью, на всю ширину. Высота фиксированная (~120px), не растягивается.

5. **Никакого пустого пространства** под шкалой — страница заканчивается сразу после шкалы.

## Фильтры на мобильном

Панель фильтров (урок, тип узла, страна, уверенность, чекбокс) — при клике на «Фильтры ▼» открывается как выдвижная панель снизу (bottom sheet) или модальное окно на весь экран. Не пытаться уместить все фильтры в горизонтальную строку на 375px — они не влезут.

## Что не менять

- Десктопный layout Explore Mode (≥769px)
- Journey Mode любой ширины
- Тёмную тему карты Explore Mode
- Логику данных и фильтров

Пришли скриншот Explore Mode на iPhone SE (375px) — должны быть видны карта на всю ширину + панель под ней + шкала внизу.
